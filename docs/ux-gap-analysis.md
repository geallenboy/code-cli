# CLI 用户体验差距分析：Code CLI vs Claude Code

## 当前状态

Code CLI 的 CLI 交互基于 `readline` + `chalk`，是一个功能完整但体验粗糙的原型。核心功能都在，但用户感知层面和 Claude Code 有明显差距。

## 逐项对比

### 1. 流式输出体验

| 维度 | Claude Code | Code CLI 现状 | 差距 |
|------|------------|--------------|------|
| 文本流式 | 逐 token 实时渲染 + Markdown 增量解析 | ✅ 逐 token 输出（`printAssistantText`） | 有基础，但无 Markdown 渲染 |
| Markdown 渲染 | `StreamingMarkdown` 增量解析，稳定前缀 memoize | ❌ 纯文本输出，无格式 | **高优先级** |
| 代码高亮 | `cli-highlight` 180+ 语言 | ❌ 无语法高亮 | **高优先级** |
| Spinner | 多模式（requesting/thinking/responding），颜色渐变 stall 指示 | ✅ 基础 Spinner（`src/ui.ts`），有 stall 检测 | 需增强模式切换 |

### 2. 工具调用透明度

| 维度 | Claude Code | Code CLI 现状 | 差距 |
|------|------------|--------------|------|
| 工具调用显示 | 实时流式参数 + 进度条 + 状态图标 | ✅ 黄色图标 + 参数摘要 | 基本够用 |
| 工具结果显示 | 完整渲染（diff 视图、代码高亮） | ⚠️ 截断到 500 字符，纯文本 | **中优先级** |
| Diff 渲染 | `structuredPatch` + 红绿着色 + 行号 | ❌ 无 diff 渲染 | **高优先级** |
| 并行工具同步闪烁 | `useBlink` 数学同步 | ❌ 无 | 低优先级（视觉细节） |

### 3. 权限确认交互

| 维度 | Claude Code | Code CLI 现状 | 差距 |
|------|------------|--------------|------|
| 确认对话框 | 组件化，风险说明 + 建议规则 + y/n/always | ✅ 基础 `Allow? [y/N]` | **中优先级** |
| 200ms 防误触 | 有 | ❌ 无 | **中优先级** |
| 风险分级显示 | LOW/MEDIUM/HIGH 颜色区分 | ✅ 有 `printPermissionRequest` 但未集成 | 需要接入 |

### 4. 输入体验

| 维度 | Claude Code | Code CLI 现状 | 差距 |
|------|------------|--------------|------|
| 多行输入 | Alt+Enter 换行 | ❌ 只支持单行 | **高优先级** |
| 输入历史 | Up/Down 浏览 + Ctrl+R 搜索 | ⚠️ readline 自带 Up/Down，无搜索 | 中优先级 |
| Tab 补全 | 文件路径 + 命令补全 | ❌ 无 | 中优先级 |
| Vim 模式 | 完整 4 模式状态机 | ❌ 无 | 低优先级 |

### 5. 会话管理

| 维度 | Claude Code | Code CLI 现状 | 差距 |
|------|------------|--------------|------|
| 会话恢复 | 自动检测未完成会话，提示恢复 | ✅ `--resume` 手动恢复 | 可改为自动提示 |
| 会话列表 | 可浏览历史会话 | ❌ 只能恢复最近一个 | 低优先级 |

### 6. 错误处理用户感知

| 维度 | Claude Code | Code CLI 现状 | 差距 |
|------|------------|--------------|------|
| 自动恢复透明 | PTL/MOT 自动恢复，用户无感 | ✅ 已实现 | 已对齐 |
| 重试进度 | 显示等待时间 + 自动重试 | ⚠️ 重试但无进度显示 | 中优先级 |
| 降级通知 | 模型降级时通知用户 | ❌ 无 | 低优先级 |

### 7. 成本与状态

| 维度 | Claude Code | Code CLI 现状 | 差距 |
|------|------------|--------------|------|
| /cost 显示 | 分模型统计 + cache read/write + 精度分级 | ✅ 基础 token + 成本 | 需增强 |
| Token bar | 每轮后显示 | ✅ `/status` 中有 | 已对齐 |
| 缓存命中率 | 显示 cache_read/cache_write | ✅ `printCacheStats` 已实现 | 已对齐 |

## 优先级排序

### P0 — 体验核心（不做就不像 CLI Agent）

1. **流式 Markdown 渲染** — 代码块高亮、标题加粗、列表缩进。当前纯绿色文本输出是最大的体验短板。
2. **Diff 渲染** — edit_file 时显示红绿 diff，让用户看到具体改了什么。
3. **多行输入** — Alt+Enter 换行，当前只能单行输入严重限制复杂 prompt。

### P1 — 体验提升（做了明显更好）

4. **权限确认增强** — 200ms 防误触 + 风险分级颜色 + 建议规则显示
5. **工具结果增强** — 代码类结果语法高亮，文件内容带行号
6. **重试进度显示** — 429/503 时显示"重试中 (2/3)..."
7. **Spinner 模式增强** — 区分 requesting/thinking/responding 状态

### P2 — 锦上添花（有时间再做）

8. **Tab 文件路径补全**
9. **Ctrl+R 历史搜索**
10. **自动会话恢复提示**
11. **Vim 模式**
12. **虚拟滚动**（长对话性能）

---

## 核心结论

当前 Code CLI 的**功能架构**已经到位（90%+ 覆盖），但**用户感知层**还停留在原型阶段。最大的 3 个体验差距是：

1. **无 Markdown 渲染** — 输出全是纯绿色文本，代码块没有高亮
2. **无 Diff 渲染** — 编辑文件后看不到具体改了什么
3. **无多行输入** — 写复杂 prompt 很痛苦

这 3 个问题解决后，CLI 体验会有质的提升。
