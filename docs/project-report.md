# AI 编程 Agent 核心技术预研报告

> Xiaomi Code — 基于 Claude Code 架构的 AI 编程 Agent 原型系统

---

## 摘要

本项目以 Anthropic 开源的 Claude Code 为研究对象，通过逆向分析其核心架构并从零实现一套功能等价的原型系统，系统性地掌握了 AI 编程 Agent 的关键技术栈。项目历时 15 个迭代周期，产出 40+ 模块、742 个自动化测试用例，实现了对 Claude Code 核心架构 90% 以上的特性覆盖。本报告阐述项目的战略定位、技术成果、能力沉淀及后续应用规划。

---

## 1. 立项背景与战略意义

### 1.1 行业趋势

AI 编程助手正在从"代码补全"向"自主编程 Agent"演进。这一转变的核心标志是：

- **GitHub Copilot** 月活用户突破 1500 万，AI 辅助编程已成为开发者基础设施
- **Claude Code** 开源后，其 Agent 架构（双层循环、工具编排、上下文工程）成为行业参考实现
- **MCP（Model Context Protocol）** 被 Anthropic 提出后，迅速获得主流 IDE 和云厂商采纳，成为 AI 工具互操作的事实标准
- 各大厂商（Microsoft、Google、字节）均在布局自有 AI Agent 平台，技术壁垒正在形成

### 1.2 项目定位

本项目**不是**要做一个 Claude Code 的替代品，而是通过"从零复现"这一工程实践方法，达成以下目标：

| 目标层级 | 具体内容 |
|---------|---------|
| **技术理解** | 深入掌握 AI Agent 的核心设计模式，理解每个架构决策背后的工程权衡 |
| **能力储备** | 建立不依赖单一供应商的 Agent 技术栈，具备自主定制和扩展的能力 |
| **资产沉淀** | 产出可复用的技术组件、完整的架构文档和团队知识库 |
| **人才培养** | 通过实战项目提升团队在 AI 工程化领域的技术深度 |

**核心逻辑**：使用商业产品解决的是当下的效率问题；掌握底层架构解决的是未来的能力问题。当行业进入 Agent 平台竞争阶段，具备自主架构能力的团队将拥有显著优势。

### 2.3 关键技术突破

**（1）双层 Generator 架构**

将 Agent 循环拆分为会话管理层（QueryEngine）和查询执行层（query），通过 async generator 连接。这一设计使得 token 追踪、预算控制、压缩触发等会话级关注点与工具调度、错误恢复、流式输出等执行级关注点完全解耦，各自独立演进。

**（2）零成本优先的压缩策略**

4 级压缩管线中，前 3 级（Snip / Micro / Collapse）均为零 API 调用的本地操作，仅在前 3 级无法释放足够空间时才触发第 4 级的模型摘要。实测中，80% 以上的压缩需求在前 3 级即可满足，显著降低了 token 消耗。

**（3）MCP 协议全链路实现**

从 stdio transport、JSON-RPC 2.0 通信、initialize 握手、工具发现（tools/list）、工具执行（tools/call），到 MCP schema → Vercel AI SDK tool 的自动转换，实现了完整的 MCP 客户端。支持多服务器并发连接和单服务器故障的优雅降级。

**（4）多 Agent 协调框架**

实现了三种协作模式：
- **Sub-Agent**：父 Agent 委派独立子任务，上下文完全隔离
- **Coordinator**：纯编排模式，4 阶段工作流（Research → Synthesize → Implement → Verify），Coordinator 本身不执行任何文件/Shell 操作
- **Swarm**：对等协作模式，Agent 间通过 Mailbox 异步通信，消息严格隔离
